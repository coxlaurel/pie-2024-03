SPRINT 2 GOAL

How can we sense and sort the marbles into eight columns?

MECHANICAL

The primary goals for the sprint were to improve team communication, finalize the marble frame design, and prototype key components such as the marble basin.

During this sprint, the team improved communication by communicating across multiple platforms like Discord, iMessages, and email. The marble frame was completed this sprint, addressing size variation among marbles with test prints and finalizing an optimal diameter of 0.5875 inches. The frame transitioned from fragile square walls to rounded grooves for better stability and closer spacing, and its size was adjusted to 8x8. This was done to simplify the overall project. The marble sorter was updated to match the new frame, adding sorters for extra columns and a trash chute.

The Archimedes screw mechanism for the marble pump was prototyped and successful at moving marble from a basin into the sorter. Additionally a solenoid mounting mechanism and chute refinements were prototyped to improve color sensing. Overall the team was able to successfully meet its goals.

ELECTRICAL

This sprint was focused on integrating and testing the marble sorter servos for full system testing as well as circuit redesign for two seperate voltage levels.

A lot of this sprint was focused on the electrical mechanical integration as the amount of new pieces being created and redesigned meant that we had to integrate find the issues and then reintegrate with the new hardware. This process ended up fixing a lot of problems that had been looked over and led to a functioning mvp for the sprint 2 review. There was also consideration in this sprint to how we would be powering the system on demo day as we needed both 5v and 16v to run the system, for this sprint we decided to use the power supplys 5v fixed to power the arduino and its components and the 16v variable to power the solenoid.

SOFTWARE

The software goal for this sprint was to write the program to sort marbles into 8 different columns based on their color. We aimed to integrate the color sensing and actuator (servo and solenoid) code to work on the two-column mechanical system first, then, scale up to the eight-column mechanical system.

We reached this software goal by finding a more accurate method of categorizing marble colors, integrating this new color sensing code with the actuator code from sprint one, and building off this integrated program to work with the eight-column mechanical system. We found that color categorization using RGB Euclidean distance worked almost all the time, compared to the manually made rules that were often incorrect, so we switched to using RGB Euclidean distance. With the color sensing functioning, we integrated it into our sprint one program and could sort marbles based on their color using the two-column mechanical system. We then updated the program to control three servo motors directing marbles into eight columns, and recalibrated the color sensor and servo motors. This allowed us to sort marbles by color using the eight-column mechanical system.
